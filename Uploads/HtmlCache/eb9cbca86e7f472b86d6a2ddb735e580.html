Topic 1. Online event management system

Problem description:

Develop a web application to manage and register for online events (workshops, courses, concerts, etc.). The system supports two types of users: participants (who register for events) and organizers (who manage events). Key functions include creating events, registering for events, searching for events, managing user information, and advanced features such as sending emails, uploading images, displaying maps, rating events, and supporting multiple languages. The application must have a user-friendly, responsive interface, ensure basic security, and be scalable.

Objectives:

-Academic: Reinforce knowledge of Java web programming, including the MVC model, HTTP request handling, database management, and basic security.

-Practical skills: Develop skills in interface design, backend programming, database integration, and teamwork using Git.

-Real-world application: Build a system simulating real-world event management, familiarizing students with commercial application development requirements.

Requirements:

1. Technologies used

-Backend: Java Spring MVC or Java Struts MVC.

-Frontend: HTML, CSS, JavaScript (recommended to use Bootstrap or JSP for responsive interfaces).

-Database: MySQL or MS SQL Server.

-Project management: Maven.

-Web server: Apache Tomcat.

-Version control: Use Git (submit source code via GitHub/GitLab).

-Additional libraries: JavaMail (for sending emails), Google Maps API (for displaying locations).

2. Main Functions

a.Users

Basic functions (45%)

-Registration/Login/Logout:

Register an account with email, password, full name, and phone number.

Log in using email and password.

-Event search:

View a list of upcoming or past events.

Search for events by name, category (workshop, course, concert), or date/time.

View event details (name, description, date/time, duration, location, available slots, organizer, event type [online/in-person/hybrid], registration deadline, ticket price [default: free], target audience, contact information).

-Event registration and Check-in:

Select an event and register (check available slots and registration deadline).

Receive a confirmation code (text-based).

Check in to the event by entering the confirmation code (verified by the system or organizer)

Cancel registration if the event has not yet occurred and the deadline allows.

-Personal information management:

Update personal information

View history of registered or attended events.

Advanced functions (10%)

-Support multilingual interface (English and Vietnamese)

-Log in using email (UEF).

-Event search: Display event location on a map (integrate Google Maps API).

-Event registration and Check-in:

Receive an email notification confirming successful registration (integrate JavaMail).

Generate and display a confirmation code as a QR code.

Allow check-in by scanning the QR code (using a web-based QR code scanner or manual code entry)

Allow rating events after participation.

b.Administrators

Basic Functions (35%)

-Event management:

Add, edit, or delete event information (name, description, date/time, duration, location, category, number of slots, event type [online/in-person/hybrid], registration deadline, ticket price [default: free], target audience, contact information).

Update event status (upcoming, ongoing, completed)..

-Event category management:

Add, edit, or delete categories (e.g., workshop, course, concert).

-Participant management:

View the list of registered participants for each event, including check-in status

View the list of registered participants for each event.

Confirm or cancel participant registrations.

Export the participant list as a CSV or Excel file.

-Reporting:

View statistics on the number of events by category, status, or event type.

View reports on the number of participants by event or time period.

Advanced Functions (10%)

-Participant management:

Send email notifications to participants when there are changes to an event or upon successful check-in.

Verify participant check-ins by scanning QR codes (using a web-based QR code scanner).

View event ratings submitted by participants.

-Reporting:

View reports on the average ratings of events.
